## GLM 四层记忆架构 (MemGPT 融合)

```
L1 Main Context (始终在 prompt)
├─ 基础人设 + Claude 记忆文件 (MEMORY.md 等)
└─ Letta 核心记忆 (human/persona)
L2 Working Memory (对话历史, ~/.glm-proxy/sessions/)
L3 Archival Memory (自动提取, ~/.glm-proxy/archival-memory.json)
L4 Vector Memory (ChromaDB 语义搜索)
```

## 踩坑

- **Daemon 崩溃**: GLM API timeout=120s 导致 daemon 180s 超时被杀, 改为 90s
- **pkill 误杀 Docker**: 用 `pkill -f 'daemon.js'` 精确匹配
- **智谱 API 余额不足**: 返回 429 1113 错误
- **codemoss config**: GUI 插件读 ~/.codemoss/config.json, 不是 cc-switch 数据库
- **Playwright MCP**: stdio 类型连接可能导致 daemon 卡死
- **Letta MCP BrokenPipeError**: 客户端断开时未处理

## MCP 配置 (全局 ~/.claude.json)

- notion (SSE) → https://mcp.notion.com/mcp
- letta-memory (SSE) → http://127.0.0.1:8788/sse
- playwright (stdio) → mcp-server-playwright

## 快捷命令

- `glm-watch` — 悬浮窗看改动
- `glm` — GLM 交互式 CLI
