**1. 配置是给 charlie 还是 root？**

NixOS 的配置是**系统级的**——对所有用户生效。但：
- 你的 `users.users.charlie` 是正常用户，有完整权限
- root 的数据（`/root/`）在 `/` 分区上，**重启不会丢**
- charlie 的数据（`/home/charlie/`）也在 `/` 分区上，**重启不会丢**
- 你现在用 root 登录的 GNOME 桌面，换 KDE 后**需要注销重新登录**才能看到 KDE

**建议你日常用 charlie 用户登录**，而不是 root。

---

**2. Docker 数据为什么总坏？**

Docker 数据存在 `/mnt/ai/docker`，而 `/mnt/ai` 是一个 **loopback 镜像文件**（`/mnt/data/ai-data.img`），挂载在 NTFS 分区上。问题链条：

```
NTFS 分区 → ai-data.img (loop) → ext4 → Docker containerd
```

如果系统非正常关机（断电/强制重启），NTFS 分区可能脏（dirty），导致 loop 镜像里的 ext4 文件系统损坏，进而 Docker 的 containerd blob 丢失。

**这很难完全避免**，但 Docker 容器本身是无状态的——数据在 volumes 里，镜像可以重新拉取。
